# CampusHotspotFix — 校园宽带热点共享修复工具

## 一、这个项目是做什么的

校园宽带（PPPoE 拨号）用户常遇到一个问题：

> PPPoE 拨号连上后，Windows 的"移动热点"开关是灰色的，或者开了热点但手机连上没网。

原因是 Windows 自带的"移动热点"界面**识别不了 PPPoE 拨号连接**（它只识别以太网/WLAN 类的"专有网络"），无法建立 NAT 转发，所以热点开了也没网。

这个工具做的就是：**绕过 Windows UI 的逻辑缺陷，直接用底层 API 把 PPPoE 拨号的网络共享给 Wi-Fi 热点。**

## 三、核心问题清单（P1–P10）

项目把所有可能遇到的问题编号为 P1–P10，代码里的 `ProblemCode` 枚举和 PRD 文档一一对应。

| 编号 | 问题 | 能否自动修 | 说明 |
|---|---|---|---|
| P1 | 移动热点开关灰色 / 驱动不支持承载网络 | ✅ | 用 `netsh wlan set/start hostednetwork` 替代 UI 创建热点 |
| P2 | 热点能开但没网（ICS 绑定失败） | ✅ | 用 COM 接口把 PPPoE 设为 ICS 公共连接 |
| P3 | WLAN 与以太网抢优先级 | ✅ | 调整网络接口跃点数 |
| P4 | 重启后配置失效 | ✅ | 注册任务计划，开机自动执行修复 |
| P5 | 约 30 分钟断开（电源/驱动超时） | ✅ | 关闭网卡省电模式 + 电源计划 |
| P6 | 约 30 分钟断开（运营商主动检测） | ⚠️ 仅诊断 | 运营商行为，工具只能告知风险 |
| P7 | 校园网 802.1x 认证冲突 | ⚠️ 仅诊断 | 认证机制导致其他设备无法接入 |
| P8 | 学校主动检测热点 | ⚠️ 仅诊断 | 政策风险，只能提示 |
| P9 | 拨号报错（651/691/628 等） | ✅ | 提供错误码对照和常见解决方法 |
| P10 | 单账号多设备限制 | ⚠️ 仅诊断 | 运营商政策，无法绕过 |

**关键设计原则**：`AutoFixable` 和 `RiskOnly` 在 UI 层面严格区分，前者给"修复"按钮，后者只展示告知文案——绝不能让用户误以为工具能解决政策性问题。

## 四、各层详解

### 4.1 Models 层 — 数据结构

定义了三组核心类型：

**`ProblemCode`（枚举）**
P1–P10 的编号，代码注释和 PRD 文档编号完全一致。比如诊断报告的日志里出现 `P2_IcsShareNotBound`，可以直接去 PRD 找到对应章节，不需要二次翻译。

**`DiagnosticItem` / `DiagnosticResult`**
一次诊断的产物——对 P1–P10 逐项检测后，每项产出一条 `DiagnosticItem`，汇总为 `DiagnosticResult`。UI 层拿到它后可以拆成两组：
- `GetAutoFixableIssues()` → 展示"一键修复"按钮
- `GetRiskOnlyIssues()` → 展示⚠️风险提示

**`FixResult`**
每个修复动作（开热点、绑 ICS、关省电……）的返回值，不是简单的 `bool`——失败时带 `ExceptionDetail`，方便日志排查。

### 4.2 Utils 层 — 工具箱

**`CommandRunner`**
统一封装所有外部命令行调用（netsh、powercfg 等）。关键处理：
- 显式指定 **GBK 编码**读取输出（中文 Windows 下 netsh 的输出默认是 GBK，不指定会乱码）
- 同时捕获 stdout 和 stderr
- 15 秒超时兜底，防止个别命令卡死整个 UI

**`AdminChecker`**
配合 `app.manifest` 的 `requireAdministrator`，做二次防御。程序启动时如果权限不足，自动弹 UAC 重新提权。

### 4.3 Services 层 — 业务逻辑

这是项目的核心，每个 Service 职责单一：

| Service | 当前状态 | 依赖 |
|---|---|---|
| `NetworkAdapterService` | ✅ 已完成 | 纯 `System.Net.NetworkInformation` 托管 API |
| `HostedNetworkService` | ✅ 已完成 | `CommandRunner`（netsh） |
| `IcsShareService` | ⏳ 待开发 | COM 互操作 `INetSharingManager` |
| `PowerManagementService` | ⏳ 待开发 | `powercfg` 命令行 + WMI |
| `TaskSchedulerService` | ⏳ 待开发 | TaskScheduler NuGet 包 |
| `DisconnectMonitorService` | ⏳ 待开发 | `HostedNetworkService.QueryStatus()` |
| `DiagnosticService` | ⏳ 待开发 | 以上所有 Service |
| `ReportService` | ⏳ 待开发 | `DiagnosticResult` + `FixResult` |

**已完成的 NetworkAdapterService 能做什么：**
- 枚举所有网络接口（以太网、WLAN、PPPoE、虚拟热点适配器等）
- 按类型筛选（`GetDialupAdapters()`、`GetHostedNetworkAdapters()`）
- 检测网卡驱动是否支持承载网络（通过 `netsh wlan show drivers` 正则匹配）

**已完成的 HostedNetworkService 能做什么：**
- 配置并启动虚拟热点（`Enable(ssid, key)`）
- 停止虚拟热点（`Disable()`）
- 查询热点运行状态和已连接客户端数量（`QueryStatus()`）

### 4.4 Forms 层 — 用户界面

当前 `MainForm` 是**最小可编译骨架**：

```
┌──────────────────────────────────┐
│  [检测承载网络]  [列出网络接口]     │
│                                  │
│  ┌──────────────────────────────┐│
│  │  诊断结果输出区域              ││
│  │  (TextBox, 只读)             ││
│  │                              ││
│  └──────────────────────────────┘│
└──────────────────────────────────┘
```

只放了两个测试按钮，目的是在这个阶段就能编译运行、验证 `NetworkAdapterService` 和 `HostedNetworkService` 的效果。

**后续完整的 UI 是三步向导**：

```
步骤 1: 诊断 → 步骤 2: 修复 → 步骤 3: 报告
  检测 P1-P10     用户选要修的项    显示结果 + 重启提示
  展示问题列表     "一键修复"按钮    可选生成报告
```

### 4.5 入口点（Program.cs）

启动流程：

```
Main()
  ├─ RegisterEncodings()   ← 加载 GBK 编码支持
  ├─ IsRunningAsAdmin()?   ← 检查管理员权限
  │   └─ No → RelaunchAsAdmin() → UAC 提权 → 重启
  ├─ --silent-fix 参数?    ← 静默模式（任务计划调用）
  │   └─ Yes → RunSilentFix() → 依次执行修复 → 退出
  └─ 正常模式 → Application.Run(new MainForm())
```

## 五、最难的部分：IcsShareService

这是整个项目技术难度最高的模块，README 也建议单独写 Console 项目先验证。

**要解决的问题**：`netsh` 没有命令可以直接配置 ICS 共享，必须走 Windows COM 接口。

**涉及的 COM 组件链**：

```
INetSharingManager              ← 入口 CLSID
  └─ INetSharingEveryConnectionCollection  ← 枚举所有连接
       └─ INetSharingEveryConnection
            └─ INetConnection   ← 包含 GUID，和 AdapterInfo.Id 一致
  └─ INetSharingConfiguration  ← 为指定连接启用/禁用共享
       ├─ EnableSharing(Public/Private)
       ├─ DisableSharing()
       └─ SharingConnectionType
```

**实现流程**：

1. 枚举所有网络连接，找到**PPPoE 适配器 GUID** → 设为 `Public`（共享出去）
2. 找到**虚拟热点适配器 GUID** → 设为 `Private`（接收共享）
3. Windows NAT 引擎自动开始转发流量

具体的 COM 接口 GUID 和 `[ComImport]` 定义在实现时会给出。

## 七、技术栈

| 层面 | 技术选型 |
|---|---|
| 语言 | C# 13 (.NET 10) |
| UI 框架 | WinForms（原生 Windows 外观，不需要额外运行时） |
| COM 互操作 | `[ComImport]` + `[Guid]` 强类型调用 |
| 命令行调用 | `System.Diagnostics.Process`，GBK 编码输出 |
| WMI 管理 | `System.Management` NuGet 包 |
| 任务计划 | `TaskScheduler` NuGet 包（David Hall） |
| 管理员权限 | `app.manifest` + `requireAdministrator` + UAC |
| 目标 OS | Windows 10 / 11 简体中文版 |

选择 WinForms 而不是 WPF / MAUI 的理由：**这个工具需要管理员权限运行，不需要花哨界面，WinForms 启动最快、依赖最少、调试最直接。**
